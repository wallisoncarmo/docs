O **Main Agents Store** é uma plataforma de **agentes de IA** que pode ser usada:
- pela **interface web** (chat e fluxos guiados), ou
- via **API** (para integrar nos seus sistemas).

Atualmente disponíveis:
- **Contestação de Infração de Trânsito** → gera **parecer jurídico em PDF** a partir de documentos obrigatórios.  
- **Litigância Abusiva** → mede **similaridade** entre petições e **estatísticas** por advogado (OAB/Nome).  
- **OCR** → extrai texto de PDFs/imagens.  
- **Transcrição + Resumo** → converte áudio em texto.

> Dica: nas páginas de *Fluxos* você pode incluir **imagens do passo a passo** da interface.

---

## Como funciona (visão geral)

**Pela Interface (UI)**
1. Escolha um **agente** (ex.: Contestação de Infração).  
2. **Envie os documentos** solicitados.  
3. Acompanhe o **status**.  
4. **Baixe o resultado** (PDF/JSON).

**Pela API**
1. `POST /agent-task/upload/{agent}/{task}` → recebe `execution_id`.  
2. `GET /agent-message/process/{execution_id}` → *polling* de status.  
3. `GET /agent-message/download/{execution_id}` → baixa o artefato final (PDF/JSON).


## Quickstart (API) — Contestação de Infração

### 1) Upload dos documentos
Documentos aceitos pelo agente `contestacao_infracao` (tarefa `contestacao`):
- `procuracao`, `crlv`, `cnh`, `alegacao2`, `ait`, `ait_radar_expirado` (opcional conforme o caso).

```bash
curl -X POST "https://agents-store-api.alfred.vert.com.br/main-agent/v1/agent-task/upload/contestacao_infracao/contestacao"   -H "Authorization: Bearer $TOKEN"   -F "procuracao=@/caminho/procuracao.pdf"   -F "crlv=@/caminho/crlv.pdf"   -F "cnh=@/caminho/cnh.pdf"   -F "alegacao2=@/caminho/alegacao2.pdf"   -F "ait=@/caminho/ait.pdf"   -F "ait_radar_expirado=@/caminho/ait_radar.pdf"
```

**Resposta**
```json
{ "execution_id": "abc123" }
```

### 2) Consultar status
```bash
curl -H "Authorization: Bearer $TOKEN"   "https://agents-store-api.alfred.vert.com.br/main-agent/v1/agent-message/process/abc123"
```

Exemplos de resposta:
```json
{ "execution_id": "abc123", "status": "PENDING" }
```
```json
{ "execution_id": "abc123", "status": "RUNNING", "message": "processando..." }
```
```json
{ "execution_id": "abc123", "status": "DONE" }
```
```json
{ "execution_id": "abc123", "status": "ERROR", "error": "detalhes..." }
```

### 3) Download do parecer (PDF)
```bash
curl -L -H "Authorization: Bearer $TOKEN"   -o parecer.pdf   "https://agents-store-api.alfred.vert.com.br/main-agent/v1/agent-message/download/abc123"
```

## Erros comuns

- **400** Arquivo ausente/inválido  
- **401** Token ausente/inválido  
- **404** `execution_id` não encontrado  
- **422** Combinação de documentos inválida para o agente  
- **500** Erro interno (tente reprocessar)

→ [Tabela completa de erros](/api-reference/errors)
